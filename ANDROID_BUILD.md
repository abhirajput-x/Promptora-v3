# Promptora ‚Äî Android app (Capacitor)

This adds a native Android wrapper around the existing Promptora React app
using [Capacitor](https://capacitorjs.com/). The website (`npm start` /
Vercel deploy) is untouched and keeps working exactly as before ‚Äî Capacitor
only wraps the already-built `build/` output in a native WebView shell.

## What was added

- `capacitor.config.json` ‚Äî Capacitor config (`appId: com.promptora.app`, `appName: Promptora`, `webDir: build`)
- `android/` ‚Äî full native Android Gradle project generated by `npx cap add android`
  (Gradle wrapper, `app` module, manifest, resources, etc.)
- New npm scripts in `package.json`:
  - `npm run cap:sync` ‚Äî copy the latest web build into the Android project
  - `npm run android:open` ‚Äî open the project in Android Studio
  - `npm run android:build-debug` ‚Äî build a debug APK from the CLI
  - `npm run android:build-release` ‚Äî build a signed release AAB from the CLI
- `android/app/promptora-release.keystore` ‚Äî an auto-generated upload keystore
  so the release `.aab` in this delivery is already signed and Play-Store
  uploadable. **Treat it as a real production secret**: back it up somewhere
  safe (a lost keystore means you can no longer publish updates to the same
  app listing). Password/alias are in `android/keystore.properties`
  (`storePassword` / `keyPassword` = `PromptoraRelease2026!`, `keyAlias` =
  `promptora`). Swap in your own keystore any time by replacing the file and
  updating `android/keystore.properties`.
- `android/app/google-services.json.example` ‚Äî Firebase placeholder (see
  `android/app/FIREBASE_SETUP.md`). Firebase is **not wired up** ‚Äî no SDK
  dependency was added ‚Äî so nothing breaks until you add your real
  `google-services.json` and Firebase dependencies later.

## Already-built outputs in this delivery

- `android/app/build/outputs/apk/debug/app-debug.apk` ‚Äî installable debug APK
- `android/app/build/outputs/bundle/release/app-release.aab` ‚Äî signed,
  Play-Store-ready Android App Bundle

## Prerequisites to rebuild yourself

- Node.js 18+
- JDK 17
- Android SDK (`ANDROID_HOME` set, with `platform-tools`, `platforms;android-34`,
  `build-tools;34.0.0` installed via `sdkmanager`)

## Rebuilding after code changes

```bash
npm install
npm run build          # builds the React app into build/
npx cap sync android   # copies build/ into android/app/src/main/assets/public
cd android
./gradlew assembleDebug     # -> app/build/outputs/apk/debug/app-debug.apk
./gradlew bundleRelease     # -> app/build/outputs/bundle/release/app-release.aab
```

## App configuration

- Package name / applicationId: `com.promptora.app`
- `minSdkVersion`: 29 (Android 10) ‚Äî matches the "Android 10+" target
- `targetSdkVersion` / `compileSdkVersion`: 34
- Environment variables (`REACT_APP_SUPABASE_URL`, `REACT_APP_SUPABASE_ANON_KEY`)
  are baked into the web build at `npm run build` time, same as the website ‚Äî
  no native-side config needed since Supabase Auth/Storage/DB calls all go
  over HTTPS from the WebView exactly like they do in the browser.
- Google OAuth: the existing Supabase-hosted OAuth flow works inside the
  Capacitor WebView as-is (it's the same HTTPS redirect flow the website
  uses). If you later want a native "Sign in with Google" button (faster,
  no browser hand-off), that requires the separate
  `@capacitor-community/google-signin` plugin ‚Äî not added here since you
  asked to keep the app as-is.

## Publishing to the Play Store

1. Create an app listing in the [Play Console](https://play.google.com/console).
2. Upload `android/app/build/outputs/bundle/release/app-release.aab`.
3. Keep `android/app/promptora-release.keystore` safe ‚Äî you'll need the same
   key (or Play App Signing) for every future update.
4. Fill in store listing, screenshots, privacy policy, content rating, etc.
   (all standard Play Console steps, unrelated to the code).

## Firebase (placeholder only, as requested)
